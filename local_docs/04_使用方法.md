# RsyncGUI 使用方法

## GitHub 与下载

- 项目地址: https://github.com/olojiang/rsync_gui
- 最新版本下载: https://github.com/olojiang/rsync_gui/releases/latest
- 全部 Releases: https://github.com/olojiang/rsync_gui/releases

从 Release 下载 `RsyncGUI-macOS.zip`，解压后将 `RsyncGUI.app` 拖入 `/Applications` 即可使用。

## 1. 环境要求

- **macOS**: 13.0 (Ventura) 或更高版本
- **Swift**: 5.9 或更高版本
- **Xcode**: 15.0 或更高版本（推荐，用于运行测试）
- **rsync**: 通常 macOS 系统已预装 `/usr/bin/rsync`

---

## 2. 构建与运行

### 2.1 命令行运行

```bash
# 进入项目目录
cd rsync_gui

# 编译
swift build

# 运行应用
swift run
```

### 2.2 构建 macOS App 包

```bash
./scripts/build_app.sh
```

构建产物为项目根目录下的 `RsyncGUI.app`，包含应用图标和 release 版本二进制。

### 2.3 Xcode 中打开（推荐）

```bash
# 生成 Xcode 项目
open Package.swift
```

或使用 Xcode 的 "Open Existing Project" 直接选择 `Package.swift` 文件。

在 Xcode 中：
1. 选择 `RsyncGUI` scheme
2. 点击 Run (Cmd+R)
3. 应用将以窗口形式启动

---

## 3. 主界面说明

```
+----------------------------------------------------------+
|  RsyncGUI                                    [+]         |
+----------------------------------------------------------+
| Profile List      |  Execution Panel                     |
|                   |                                      |
| [Documents]  ✏️   |  Documents                           |
| [Photos]     ✏️   |  [●] 成功                            |
|                   |                                      |
|                   |  来源: /Users/me/Documents           |
|                   |  目标: /Users/me/Backups/Documents   |
|                   |  选项: -avz --delete                 |
|                   |  命令: rsync -avz --delete ...       |
|                   |                                      |
|                   |  [执行] [取消] 清空日志              |
|                   |                                      |
|                   |  ----------------------------------  |
|                   |  [10:00:01.234] [INFO] sending ...   |
|                   |  [10:00:01.235] [INFO] sending ...   |
|                   |  [10:00:02.100] [INFO] done          |
+----------------------------------------------------------+
```

### 3.1 左侧 — 配置列表

- **选择配置**: 单击列表项，右侧展示详情
- **新建配置**: 点击工具栏 `+` 按钮
- **编辑配置**: 点击列表项右侧的铅笔图标
- **删除配置**: 右键点击列表项，选择「删除」

### 3.2 右侧 — 执行面板

- **状态徽章**: 圆点颜色表示执行状态
  - 🔵 蓝色 = 执行中
  - 🟢 绿色 = 成功
  - 🔴 红色 = 失败
  - 🟠 橙色 = 已取消
- **配置详情**: 展示来源、目标、选项、完整命令
- **执行按钮**: `Cmd+R` 快捷键触发
- **取消按钮**: 终止正在执行的 rsync 进程
- **清空日志**: 清除当前执行日志
- **执行日志文件**: 每次执行都会写入 `~/Library/Logs/RsyncGUI/*.log`，界面会显示日志路径，并提供 Finder 打开按钮
- **目录校验**: 执行前会检查来源和目标目录是否存在；任一目录不存在时不会启动 rsync
- **脚本目录**: 可配置生成脚本的保存目录；目录不存在时生成脚本会自动创建
- **生成脚本**: 保存后会在 Finder 中打开并选中生成的脚本文件

## 4. 创建配置

### 4.1 步骤

1. 点击左上角 `+` 按钮，弹出编辑窗口
2. 填写**配置名称**（如 "Documents Backup"）
3. 点击**来源**行的「浏览」按钮，选择要同步的源目录
4. 点击**目标**行的「浏览」按钮，选择同步目标目录
5. 选择预设或填写 **rsync 选项**（空格分隔，如 `-a --human-readable --info=progress2`）
6. 点击「保存」

### 4.2 常用 rsync 选项参考

| 选项 | 含义 |
|------|------|
| `-a` | 归档模式（保留权限、时间戳、符号链接等） |
| `-v` | 详细输出，显示每个传输的文件 |
| `-z` | 传输时压缩数据 |
| `--delete` | 删除目标中源不存在的文件（使目标与源完全一致） |
| `--dry-run` | 模拟运行，不实际执行任何操作 |
| `--progress` | 显示传输进度 |
| `--info=progress2` | 显示整体传输进度，适合 GUI 日志 |
| `--human-readable` | 使用更易读的文件大小单位 |
| `--backup` | 覆盖目标文件前保留旧版本 |
| `--partial` | 保留中断传输的部分文件 |
| `-e ssh` | 通过 SSH 隧道传输（用于远程同步） |

**示例组合**:
```
-a --human-readable --info=progress2
```

## 5. 执行同步

1. 在左侧列表中选择要执行的配置
2. 点击右侧的「执行」按钮（或按 `Cmd+R`）
3. 观察日志控制台中的实时输出
4. 执行完成后，状态徽章会更新为「成功」或「失败」

### 5.1 取消执行

执行过程中，点击「取消」按钮会发送终止信号给 rsync 进程，状态变为「已取消」。

关闭应用窗口或退出应用时，也会同步终止当前仍在运行的 rsync 子进程，避免界面消失后后台继续传输。

## 6. 日志阅读

日志控制台展示所有 rsync 的标准输出和错误输出：

```
[10:00:01.234] [INFO] sending incremental file list
[10:00:01.235] [INFO] docs/
[10:00:01.236] [INFO] docs/report.pdf
[10:00:02.100] [INFO] done
```

- **时间戳**: 精确到毫秒，便于定位问题发生时刻
- **级别**: INFO（标准输出）、ERROR（标准错误）
- **颜色**: INFO 为默认色，ERROR 为红色
- **选中**: 支持鼠标选中并复制日志文本
- **自动滚动**: 新日志追加和任务结束时会自动滚动到底部

日志文件保存到：

```
~/Library/Logs/RsyncGUI/
```

如果界面异常关闭或任务失败，可以从该目录找到完整执行记录。

---

## 7. rsync 版本选择

应用会按顺序选择可执行的 rsync：

1. `/opt/homebrew/bin/rsync`
2. `/usr/local/bin/rsync`
3. `/usr/bin/rsync`

macOS 自带 `/usr/bin/rsync` 通常较旧，可能不支持 `--info=progress2`。如果只找到旧版 rsync，应用会自动把该选项降级为 `--progress`，并在界面日志和 `.log` 文件中记录原因。

---

## 8. 配置持久化

所有配置自动保存到：

```
~/Library/Application Support/RsyncGUI/profiles.json
```

应用重启后，配置会自动加载。可以直接备份或迁移此 JSON 文件。
